ฟีเจอร์: ระบบบันทึกกิจกรรมระบบ (System Logs)

1. วัตถุประสงค์
    ฟีเจอร์นี้เป็นศูนย์รวมในการดูข้อมูลกิจกรรมต่าง ๆ ของผู้ใช้ภายในระบบอย่างละเอียด โดยปฏิบัติตามมาตรฐานการจัดเก็บข้อมูลจราจรทางคอมพิวเตอร์และ PDPA เพื่อใช้ในการตรวจสอบ สิทธิ์การเข้าถึง และสอดส่องความผิดปกติของระบบ

2. สิ่งที่ควรทราบก่อนใช้งาน
    - ฟีเจอร์นี้สงวนไว้สำหรับผู้ใช้งานระดับ ผู้ดูแลระบบ (Admin) เท่านั้น
    - ระบบจะบันทึกข้อมูลแบบเบื้องหลัง (Background process) ทุกครั้งที่มีผู้ใช้งานเข้าสู่ระบบ (Login/Logout) สร้าง แก้ไข หรือลบข้อมูลสำคัญ
    - ตามนโยบายการจัดเก็บข้อมูล ข้อมูลใน System Logs จะถูกเก็บไว้เป็นระยะเวลาสูงสุด 90 วัน จากนั้นระบบจะทำการลบข้อมูลที่เก่าเกินกว่าระยะเวลาที่กำหนดโดยอัตโนมัติแบบ Hard Delete

3. ขั้นตอนการใช้งานระบบ System Logs
    Step 1: เข้าสู่ระบบ
        เข้าสู่ระบบด้วยบัญชีผู้ดูแลระบบ (Admin)
    Step 2: ไปที่หน้า System Logs
        ไปที่ Dashboard
        เลือกเมนู “System Logs”
    Step 3: การค้นหาและคัดกรองข้อมูล (Filter)
        - สามารถค้นหาข้อมูลที่ต้องการได้ในแถบเครื่องมือค้นหาด้านบน
        - ค้นหาแบบทั่วไป: พิมพ์คำค้นหา เช่น Username, Email หรือ IP Address ลงในช่องค้นหา ระบบจะค้นหาข้อมูลที่ตรงกันแบบ (Case-insensitive)
        - คัดกรองตามกิจกรรม (Action): เลือกประเภทกิจกรรมจาก Dropdown menu เช่น LOGIN, CREATE_DATA, DELETE_DATA หรือดูทั้งหมด (ALL)
        - คัดกรองตามวันที่: เลือกช่วงเวลาที่ต้องการตรวจสอบผ่านปฏิทินที่กำหนด
        - คลิกปุ่ม "ค้นหา" เพื่อดูผลลัพธ์ ข้อมูลจะแสดงรายละเอียด ได้แก่ เวลา (Timestamp), USER/IP, ประเภทกิจกรรม (Action) และรายละเอียด (Details)
    Step 4: การตรวจสอบรายละเอียด (Details)
        - ในตารางแสดงผล สามารถคลิกดูรายละเอียดของ Log แต่ละรายการเพื่อดูข้อมูลเชิงลึกในรูปแบบ JSON
        - ข้อมูลนี้จะแสดงรายละเอียดเพิ่มเติมเกี่ยวกับการกระทำ เช่น รหัสผู้ใช้งาน, ประเภทระบบ (Resource), และข้อความอธิบายการเปลี่ยนแปลง
    Step 5: การส่งออกข้อมูล (Export Logs)
        - หากต้องการนำข้อมูลออกเพื่อเป็นหลักฐานหรือทำรายงาน ให้คลิกที่ปุ่มไอคอน "Export" (ปุ่มสีส้ม) หรือปุ่มดาวน์โหลดที่หน้า System Logs
        - ระบบจะทำการดาวน์โหลดไฟล์ในรูปแบบ JSON พร้อมกับแนบรหัสเข้ารหัสความปลอดภัย (SHA-256 Signature) เพื่อป้องกันการแก้ไขข้อมูล

4. กรณีเกิดปัญหาและข้อควรระวัง
    - ไม่พบข้อมูล System Logs: ระบบอาจจะไม่มีกิจกรรมที่ตรงกับเงื่อนไข หรือกรองช่วงเวลาผิด ลองเคลียร์ตัวกรองการค้นหาแล้วลองใหม่อีกครั้ง
    - คำเตือนในไฟล์ที่ Export ออกมา: ไฟล์ที่ถูกส่งออก (Export) ในส่วนของข้อมูลดิบ (data) ห้ามทำการแก้ไขเด็ดขาด เพราะจะทำให้รหัส (SHA-256) ป้องกันความถูกต้องใน metadata ไม่ตรงกัน และถือว่าไฟล์อาจถูกดัดแปลง (Data Tampering)
